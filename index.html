Première partie

Le Pseudo-code

Qu'est-ce que le pseudo-code ?

  • 📝 Une manière simple de décrire des étapes.
  • ❌ Ce n'est pas un vrai langage de programmation.
  • ✅ Sert uniquement à représenter une logique.

Les 2 types d'exercices

1. Trouver le résultat attendu

Déterminez la valeur finale affichée.

A ← 5; B ← 2;
Afficher A+B

Résultat attendu : 7

2. Code manquant

Complétez l'instruction manquante.

Si A > 5 Alors
  [ ??? ]

Résultat attendu : Afficher "Bravo"

📦 Variable

Une boîte nommée qui stocke une valeur en mémoire.

age → 25
score → 100

💡 Règle : garde sa DERNIÈRE valeur.

⬅️ Affectation

Syntaxe : variable ← valeur

"Mettre la valeur de DROITE dans la boîte de GAUCHE"

x ← 5
x ← x + 3
y ← x * 2
💾 Mémoire

⚠️ Le ← n'est PAS une équation mathématique !
x ← x + 1 signifie : x prend sa valeur actuelle + 1.

🖥️ Afficher

Texte

afficher "Bonjour"

→ Bonjour

Variable

x ← 42
afficher x

→ 42

Ordre

afficher 1
afficher 2

→ 1
→ 2

💡 Règle d'or : chaque instruction afficher = une nouvelle ligne. L'ordre du code = ordre d'affichage.

🔗 Concaténation

Coller des textes et des variables ensemble.

"Bonjour" + " " + "Ali" → "Bonjour Ali"

Texte + Variable

score ← 95
msg ← "Score : " + score
afficher msg
→ Score : 95

Construire pas-à-pas

code ← "ID"
code ← code + "-"
code ← code + "42"
afficher code
→ ID-42

⚠️ Un espace doit être explicitement ajouté : " "
Sans espace: "Ali" + "Ben" = "AliBen"

➗ MOD — Reste de Division

Analogie : 17 pizzas pour 5 personnes.

17 DIV 5 = 3
Quotient (chacun a 3)
17 MOD 5 = 2
Reste (il reste 2 pizzas)
Parité :
n MOD 2 = 0
→ pair
Dernier chiffre :
n MOD 10
→ dernier chiffre
Cyclique :
i MOD 3
→ 0, 1, 2, 0, 1...

🔀 SI / SINON — Conditions

si condition alors
   // bloc si vrai
sinon
   // bloc si faux
fin si

Un cheminement logique basé sur une décision.

💡 Indentation = niveau d'imbrication.

Animation : Trace SI / SINON

age ← 20
SI age >= 18 ALORS
  AFFICHER "Majeur"
SINON
  AFFICHER "Mineur"
FIN SI
💾 Mémoire
🖥️ Écran

⚖️ Les Comparaisons

> Plus grand
< Plus petit
>= Plus grand ou égal
<= Plus petit ou égal
== Égal à
!= Différent de

🔗 Opérateurs Logiques

ET (AND)

Toutes les conditions doivent être vraies.

OU (OR)

Au moins une condition doit être vraie.

NON (NOT)

Inverse la condition (Vrai devient Faux).

🔁 Les Boucles

POUR / TANT QUE

Répéter une action plusieurs fois de manière automatique.

pour i de 1 a 3 faire
   afficher i
fin pour
i=1 → 1
i=2 → 2
i=3 → 3

🔄 Boucles Imbriquées (Animation)

POUR i DE 1 À 2 FAIRE
  POUR j DE 1 À 2 FAIRE
    AFFICHER i + "-" + j
  FIN POUR
FIN POUR
💾 Mémoire
🖥️ Écran

Comment lire :

  1. Boucle EXTERNE : i démarre à 1.
  2. Boucle INTERNE : j parcourt 1 puis 2.
  3. La boucle interne finit → i avance, et j repart TOUJOURS de 1.

🧩 Un exemple complet (Animation)

score ← 0
POUR i DE 1 À 3 FAIRE
  SI i MOD 2 == 0 ALORS
    score ← score + 10
  SINON
    score ← score + 5
  FIN SI
FIN POUR
AFFICHER "Score final : " + score
💾 Mémoire
🖥️ Écran
(i=1 → impair → +5)
(i=2 → pair → +10)
(i=3 → impair → +5)

✅ Conseils pour réussir

  • 📖 Lire calmement chaque exercice.
  • 👁️ Observer tous les détails.
  • ⏳ Prendre le temps de réfléchir.
  • 🪜 Procéder étape par étape.
  • 🎲 Ne pas répondre au hasard.
Insistez sur la gestion du stress. C'est un test de logique, la précipitation est l'ennemi.

❌ Les erreurs fréquentes

  • 💨 Lire trop vite les consignes.
  • 🙈 Ignorer une information clé.
  • 🏃 Se précipiter pour finir vite.
  • 🔄 Modifier une réponse sans raison logique.
  • 😰 Se laisser envahir par le stress.
